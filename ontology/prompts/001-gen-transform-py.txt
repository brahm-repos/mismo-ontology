MISMO_3.6.0_B367.xsd is the mismo standard XSD which defines mortgage industry entities, 
attributes etc. Your task is to transform this XSD into a RDF/RDFS/OWL in turtle format. 
Follow the below instructions, guidelines and pattern based transformation to transform the
MISMO XSD into a OWL in turtle format. These are numbered starting with 1.


1. Standard Prefixes:
Please add the following prefixes at the begining of the TTL file.

@prefix mismo: <http://www.mismo.org/residential/2009/schemas#> .
@prefix owl: <http://www.w3.org/2002/07/owl#> .
@prefix rdf: <http://www.w3.org/1999/02/22-rdf-syntax-ns#> .
@prefix rdfs: <http://www.w3.org/2000/01/rdf-schema#> .
@prefix skos: <http://www.w3.org/2004/02/skos/core#> .
@prefix xsd: <http://www.w3.org/2001/XMLSchema#> .


2. Read the XSD such that you take one completer type at a time and process.

Examples of complete type are:
Example 01:
    <xsd:simpleType name="MISMOEnum_Base">
        <xsd:restriction base="xsd:string">
            <xsd:maxLength value="80"/>
        </xsd:restriction>
    </xsd:simpleType>

Example 02:
    <xsd:complexType name="ALIASES">
        <xsd:annotation>
            <xsd:documentation>Information about one or more ALIAS. Holds all occurrences of ALIAS.</xsd:documentation>
        </xsd:annotation>
        <xsd:sequence>
            <xsd:element name="ALIAS" type="ALIAS" minOccurs="0" maxOccurs="unbounded">
                <xsd:annotation>
                    <xsd:documentation>Other names that the borrower may be known by and associated information.</xsd:documentation>
                </xsd:annotation>
            </xsd:element>
            <xsd:element name="EXTENSION" type="ALIASES_EXTENSION" minOccurs="0"/>
        </xsd:sequence>
        <xsd:attributeGroup ref="xlink:MISMOresourceLink">
            <xsd:annotation>
                <xsd:documentation>An xlink attribute group for defining the end point of a relationship.</xsd:documentation>
            </xsd:annotation>
        </xsd:attributeGroup>
        <xsd:attributeGroup ref="AttributeExtension">
            <xsd:annotation>
                <xsd:documentation>An implementation of the ability to extend an instance document by using any attribute. To change from Lax to make an redefinition restriction of AttributeExtension attribute group.</xsd:documentation>
            </xsd:annotation>
        </xsd:attributeGroup>
    </xsd:complexType>


3. Pattern 001

Sample XSD Pattern:
    <xsd:simpleType name="MISMOEnum_Base">
        <xsd:restriction base="xsd:string">
            <xsd:maxLength value="80"/>

Sample TTL Transformation

mismo:MISMOEnum_Base a rdfs:Datatype ;
    rdfs:label "MISMOEnum_Base" ;
    rdfs:comment "String datatype with maximum length of 80 characters" ;
    rdfs:subClassOf xsd:string .

Simple XSD Pattern
    <xsd:simpleType name="MISMODate_Union">
        <xsd:union memberTypes="xsd:gYear xsd:gYearMonth xsd:date"/>

# MISMODate_Union - Union type combining different date formats
mismo:MISMODate_Union a rdfs:Datatype ;
    rdfs:label "MISMODate_Union" ;
    rdfs:comment "A union datatype that accepts year (gYear), year-month (gYearMonth), or full date (date) formats" ;
    rdfs:subClassOf xsd:gYear, xsd:gYearMonth, xsd:date .

Sample XSD Pattern

        <xsd:restriction base="MISMOEnum_Base">
            <xsd:enumeration value="NotCollected">
                <xsd:annotation>

                </xsd:annotation>
            </xsd:enumeration>
            <xsd:enumeration value="NotReceived">
                <xsd:annotation>
                    <xsd:documentation>The creator of the MISMO message can collect the data point, but has not received a value as of the time the file was created.</xsd:documentation>
                </xsd:annotation>
            </xsd:enumeration>
            <xsd:enumeration value="NotRelevant">
                <xsd:annotation>
                    <xsd:documentation>The creator of the MISMO message can collect the data point, but did not ask for it because it is not germane to the context at hand. (One example is “cash out purpose type” for a purchase loan.)</xsd:documentation>
                </xsd:annotation>
            </xsd:enumeration>
            <xsd:enumeration value="Omitted">
                <xsd:annotation>
                    <xsd:documentation>The creator of the MISMO message has a value for the data point. However, they have a pertinent reason (such as confidentiality) for not including it in the message.</xsd:documentation>
                </xsd:annotation>
            </xsd:enumeration>
            <xsd:enumeration value="Other"/>
        </xsd:restriction>
    </xsd:simpleType>

mismo:MISMOEnum_Base a rdfs:Datatype ;
